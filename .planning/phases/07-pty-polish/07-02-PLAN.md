    1. Add import at top:
       `import { isInteractiveCommand, executeInteractive } from './interactive.js'`

    2. Update initial state to include `interactiveRunning: false` in the ShellState literal (after `aiStreaming: false`)

    3. Update the SIGINT handler (rl.on('SIGINT')) per D-15:
       Add at the TOP of the handler, before the aiStreaming check:
       ```
       if (state.interactiveRunning) {
         return  // Let child process handle Ctrl+C
       }
       ```

    4. Update the `case 'passthrough'` block. BEFORE the existing `executeCommand` call, add interactive detection per D-05, D-06, D-12, D-13, D-16, D-17:
       ```
       const interactiveList = config.interactive_commands ?? []
       if (isInteractiveCommand(action.command, interactiveList)) {
         // Per D-05, D-12: Pause readline to release stdin ownership
         rl.pause()
         state = { ...state, interactiveRunning: true }

         const interactiveResult = await executeInteractive(action.command)

         // Per D-16: Reset ANSI attributes
         process.stdout.write('\x1b[0m')
         // Per D-17: Ensure fresh line
         process.stdout.write('\n')
         // Per D-06, D-13: Resume readline
         state = { ...state, interactiveRunning: false }
         rl.resume()

         if (interactiveResult.exitCode !== 0) {
           process.stderr.write(`[exit: ${interactiveResult.exitCode}]\n`)
           state = { ...state, lastError: { command: action.command, stderr: '', exitCode: interactiveResult.exitCode }, lastSuggestedFix: undefined }
         } else {
           state = { ...state, lastError: undefined, lastSuggestedFix: undefined }
         }
         break
       }
       ```
       The existing `executeCommand` path remains as the `else` fallthrough for non-interactive commands.

    5. Every spread of state must include interactiveRunning implicitly (it's already in the interface and initial state, so spreads carry it forward). No other changes needed.

    Note on D-12 raw mode: Do NOT explicitly call `process.stdin.setRawMode(false/true)`. Per research Pitfall #5, readline manages raw mode internally. `rl.pause()` and `rl.resume()` handle it. Adding explicit setRawMode calls risks desync.
