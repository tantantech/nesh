  <action>
    Create `src/plugin-manager.ts` per D-17 through D-27:

    Export interface for shell.ts integration:
    ```
    export interface PluginManagerContext {
      readonly pluginRegistry: PluginRegistry
      readonly rl: readline.Interface
      readonly onHotReload?: (result: HotReloadResult) => void
    }
    export interface HotReloadResult {
      readonly registry: PluginRegistry
      readonly hookBus: HookBus
      readonly enabled: readonly PluginManifest[]
    }
    export async function executePlugin(args: string, ctx: PluginManagerContext): Promise<void>
    ```

    Implement dispatcher that parses first word of args as subcommand:

    **`list`** (D-19): Get BUNDLED_PLUGINS + any external plugins. For each, show name, version, [enabled]/[disabled], platform annotation ([macos]/[linux]/[all]) per D-39. Use `pc.green` for enabled, `pc.dim` for disabled.

    **`enable <name>`** (D-20): Load config, add name to `plugins.enabled` array (if not already present), save config. Print confirmation. Signal hot-reload via `ctx.onHotReload` callback (actual reload logic in Plan 04).

    **`disable <name>`** (D-20): Load config, remove name from `plugins.enabled` array, save config. Print confirmation. Signal hot-reload.

    **`search <query>`** (D-24): Filter BUNDLED_PLUGINS where `name.includes(query) || description.includes(query)` (case-insensitive). Display matching plugins with descriptions.

    **`doctor`** (D-25): Call `pluginRegistry.getPlugins()`, show any with status 'failed'. Show total plugin count, enabled count, list missing dependencies. Recommend `nesh --safe` if many failures.

    **`times`** (D-26): Show that Phase 1 sync loading completed (timestamp from loader). Per-plugin timing requires instrumentation -- show enabled plugin count and note that individual timing is available in verbose mode.

    **`profile`** (D-16, D-14, D-15): Import PROFILES and expandProfile from `./plugins/profiles.js`. Show numbered menu of profiles with descriptions and included plugins. On selection, call `expandProfile(name)`, write result to `config.plugins.enabled`, call `saveConfig`. Use the readline question pattern from settings.ts per D-27.

    **`help` / no args**: Print usage showing all available subcommands.

    Keep functions under 50 lines. Split subcommands into separate functions within the same file. Use `process.stdout.write` (no console.log).
