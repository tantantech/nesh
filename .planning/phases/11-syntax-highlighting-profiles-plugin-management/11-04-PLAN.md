    - installPlugin returns the loaded manifest on success, null on failure
    - installPlugin warns user before installing (security notice per D-36)
    - installPlugin rejects if git is not available
    - installPlugin rejects if plugin directory already exists

    - updatePlugin returns error string if directory does not exist
    - removePlugin('name') removes directory and removes from config.plugins.enabled
  </behavior>
  <action>
    Create `src/plugin-install.ts` per D-21, D-22, D-23, D-32, D-33, D-36:

    Import `spawn` from `node:child_process`, `fs` from `node:fs`, `path` from `node:path`, `CONFIG_DIR`, `loadConfig`, `saveConfig` from `./config.js`, `loadExternalPlugin` from `./plugins/external.js`.

    `const PLUGINS_DIR = path.join(CONFIG_DIR, 'plugins')`

    Export `async function installPlugin(repoRef: string, rl: readline.Interface): Promise<PluginManifest | null>`:
    1. Parse repoRef: if `user/repo` format, expand to `https://github.com/user/repo.git`; if full URL, use as-is
    2. Extract plugin name from repo (last path segment without .git)
    3. Check `PLUGINS_DIR/<name>` does not already exist -- if it does, print error and return null
    4. Print security warning: "Installing external plugin from <url>. External plugins run with full system access."
    5. Ask confirmation via `rl.question('Continue? (y/N): ')`; if not 'y'/'Y', return null
    6. Ensure `PLUGINS_DIR` exists (`fs.mkdirSync(PLUGINS_DIR, { recursive: true })`)
    7. Spawn `git clone --depth 1 <url> <PLUGINS_DIR>/<name>` -- await completion
    8. On clone failure, print error with stderr, clean up partial directory, return null
    9. Call `loadExternalPlugin(pluginPath)` to validate manifest
    10. If manifest invalid, print error, remove directory, return null
    11. Print plugin name, version, description from manifest
    12. Return manifest

    Export `async function updatePlugin(name: string): Promise<string | null>`:
    1. Check `PLUGINS_DIR/<name>` exists -- if not, return error string
    2. Spawn `git -C <pluginDir> pull --ff-only` -- await completion
    3. On failure, return error string with stderr
    4. Return null (success)

    Export `async function removePlugin(name: string): Promise<void>`:
    1. Remove `PLUGINS_DIR/<name>` directory recursively (`fs.rmSync(dir, { recursive: true, force: true })`)
    2. Load config, remove name from `plugins.enabled` if present, save config

    Helper `function spawnAsync(cmd: string, args: string[], opts?: SpawnOptions): Promise<{ code: number; stderr: string }>` -- wraps spawn in a Promise.

    Helper `function isGitAvailable(): Promise<boolean>` -- spawn `git --version`, return true if exit 0.

    Create tests in `tests/plugin-install.test.ts` -- test parsing, directory checks, config updates (mock fs and spawn).

    Import types: `PluginRegistry` from `./plugins/registry.js`, `HookBus` from `./plugins/hooks.js`, `PluginManifest` from `./plugins/types.js`.

    Export interface:
    ```
    export interface HotReloadResult {
      readonly registry: PluginRegistry
      readonly hookBus: HookBus
      readonly enabled: readonly PluginManifest[]
    }
    ```

    Export `async function hotReload(): Promise<HotReloadResult>`:
    1. `const freshConfig = loadConfig()` -- re-read config from disk (enable/disable already saved it)
    2. `const external = await discoverExternalPlugins()` -- load any external plugins
    3. `const allPlugins = [...BUNDLED_PLUGINS, ...external]` -- combine bundled + external
    4. `const phase1 = loadPluginsPhase1(freshConfig.plugins ?? {}, allPlugins)` -- rebuild registry from scratch per D-30
    5. `const newHookBus = buildHookBus(phase1.enabledPlugins)`
    6. Fire-and-forget Phase 2 async init for newly-enabled plugins: `setImmediate(async () => { await loadPluginsPhase2(phase1.enabledPlugins, { cwd: process.cwd() }) })`
    7. Return `{ registry: phase1.registry, hookBus: newHookBus, enabled: phase1.enabledPlugins }`

    This module is pure orchestration -- no tests needed beyond integration (type-checked via tsc). The underlying functions (loadPluginsPhase1, buildHookBus) are already tested.
