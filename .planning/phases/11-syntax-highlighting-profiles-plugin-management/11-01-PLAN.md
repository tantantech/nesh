Build the syntax highlighting engine: a pure tokenizer that classifies shell input tokens, a command validity cache using compgen -c, and an ANSI renderer that overwrites the visible line without touching rl.line.

Purpose: Enable real-time input coloring so users see valid commands in green, invalid in red, strings in yellow, flags in cyan -- making the shell feel professional and helping catch typos before hitting Enter.

Output: Three modules in src/highlighting/ ready to be wired into shell.ts keypress events (wiring happens in Plan 05).
</objective>

    - tokenize("ls | grep foo && echo done") splits on operators: command, operator, command, argument, operator, command, argument
    - tokenize("git commit -m \"msg\"") handles double-quoted strings with flags
    - tokenize("") returns empty array

    - isKnownCommand returns true after refreshCommandCache populates the set

  <action>
    Create `src/highlighting/tokenizer.ts` per D-02 and D-07:
    - Export `TokenType = 'command' | 'command-invalid' | 'flag' | 'string' | 'path' | 'operator' | 'argument'`
    - Export `interface Token { readonly type: TokenType; readonly value: string; readonly start: number }`
    - Export `function tokenize(line: string, isKnown: (cmd: string) => boolean): readonly Token[]`
    - Split on shell operators (`|`, `&&`, `||`, `;`) using regex `/(\|{1,2}|&&|;)/` -- operators become `operator` tokens
    - In each segment, first word = command (check via `isKnown` callback for valid/invalid)
    - Remaining words: starts with `-` -> flag, quoted (`'` or `"`) -> string (scan for closing quote), contains `/` -> path, else -> argument
    - Track `start` offset for each token by walking the original line
    - Pure function, no side effects, no I/O

    Create `src/highlighting/commands.ts` per D-04:
    - Module-level `let knownCommands: Set<string> = new Set()` and `let lastRefresh = 0`
    - `REFRESH_INTERVAL_MS = 60_000`
    - Export `async function refreshCommandCache(): Promise<void>` -- spawns `bash -c 'compgen -c'`, collects stdout, populates knownCommands Set. Silent on error (resolve, don't reject). Use `spawn` from `node:child_process`.
    - Export `function isKnownCommand(name: string): boolean` -- returns `knownCommands.has(name)`. If `Date.now() - lastRefresh > REFRESH_INTERVAL_MS`, fire-and-forget `refreshCommandCache()` (non-blocking).
    - Export `function addKnownCommands(names: Iterable<string>): void` -- merges plugin aliases into the known set (called during setup with registry alias keys)

    Create tests in `tests/highlighting/tokenizer.test.ts` using vitest.

    - renderHighlighted writes colored line to stdout and restores cursor position
    - renderHighlighted is a no-op when stdout is not TTY
    - clearHighlighting writes erase sequence

  </behavior>
  <action>
    Create `src/highlighting/renderer.ts` per D-03, D-06, D-08, D-10:
    - Import `picocolors` (pc) for colors, `moveCursor` from `node:readline`, `tokenize` from `./tokenizer.js`, `isKnownCommand` from `./commands.js`
    - Export `function colorize(tokens: readonly Token[]): string` -- maps each token to its colored version using pc.green (command), pc.red (command-invalid), pc.cyan (flag), pc.yellow (string), pc.blue (path), pc.magenta (operator), default for argument. Joins with spaces preserving original whitespace.
    - Export `function renderHighlighted(rl: { readonly line: string; readonly cursor: number }, isKnown: (cmd: string) => boolean): void`:
      1. If `!process.stdout.isTTY` return early
      2. `const start = performance.now()`
      3. Tokenize `rl.line` using `tokenize(rl.line, isKnown)`
      4. Build colored string via `colorize(tokens)`
      5. Check `performance.now() - start > 16` -- if exceeded, return (plain text fallback per D-06)
      6. Move cursor back by `rl.cursor` columns (to start of typed content)
      7. Write colored string
      8. Compute visible length (strip ANSI with `/\x1b\[[^m]*m/g`), move cursor to restore position
    - Export `function clearHighlighting(): void` -- writes `\x1b[K` erase to end of line
    - CRITICAL: Use visible character length (strip ANSI) for all moveCursor arithmetic, never byte length
