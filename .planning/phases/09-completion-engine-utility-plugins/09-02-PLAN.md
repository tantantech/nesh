    - Engine uses compgen for file completion when command position is not first word and no provider exists
    - Engine wraps provider calls in Promise.race with 1000ms timeout (D-05)
    - Engine caches results per (commandName:currentWord) with 30s TTL (D-06)
    - Engine returns cached results on repeated calls with same input
    - Engine returns [[], line] on empty input
    - Engine never throws -- all errors caught and return empty results
    - createCompletionEngine returns object with complete(line) method
  </behavior>
  <action>
    Create `src/completions/engine.ts` per D-09, D-10, D-11:
    - Export `createCompletionEngine(registry: PluginRegistry)` returning `{ complete(line: string): Promise<[string[], string]> }`
    - The `complete` function:
      1. Call `parseCompletionContext(line)` to get context
      2. Build cache key: `${context.commandName}:${context.currentWord}`
      3. Check cache -- if hit, return `[cached.items as string[], cached.prefix ?? context.currentWord]`
      4. Resolution order (per D-10):
         a. `registry.getCompletionProvider(context.commandName)` -- if exists, call with 1s timeout via `Promise.race`
         b. `registry.getCompletionSpecs(context.commandName)` -- if exists, get first spec and call `resolveFromSpec(spec, tokens, currentWord, context)`
         c. Fallback: if first word position, `compgenComplete('command', currentWord)`; else `compgenComplete('file', currentWord)`
      5. Cache result with 30_000ms TTL if items.length > 0
      6. Return `[result.items as string[], result.prefix ?? context.currentWord]`
    - All provider/spec/compgen calls wrapped in try/catch -- on error return `{ items: [], prefix: context.currentWord }`
    - The 1-second timeout: `Promise.race([providerCall, new Promise<CompletionResult>(resolve => setTimeout(() => resolve({ items: [] }), 1000))])`
    - Import from: `./types.js`, `./cache.js`, `./compgen.js`, `./spec-parser.js`, `../plugins/registry.js`
