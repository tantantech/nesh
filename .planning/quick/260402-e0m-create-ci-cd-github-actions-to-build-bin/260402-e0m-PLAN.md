<task type="auto">
  <name>Task 1: Create Node.js SEA binary build script</name>
  <files>scripts/build-binaries.sh</files>
  <action>
Create `scripts/build-binaries.sh` that builds a standalone Node.js Single Executable
Application (SEA) binary. The script should:

1. Accept a target platform argument (or default to current platform).
2. Run `npm run build` first to produce `dist/cli.js`.
3. Generate a SEA config JSON (`sea-config.json`) pointing at `dist/cli.js` with:
   - `"main": "dist/cli.js"`
   - `"output": "sea-prep.blob"`
   - `"disableExperimentalSEAWarning": true`
   - `"useSnapshot": false`
   - `"useCodeCache": true`
4. Run `node --experimental-sea-config sea-config.json` to generate the blob.
5. Copy the current Node binary to `claudeshell-{platform}-{arch}` (e.g., `claudeshell-macos-arm64`).
6. Use `npx postject` to inject the SEA blob into the copied binary:
   - For macOS: `npx postject claudeshell-{name} NODE_SEA_BLOB sea-prep.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6ac44c7517b7c16b6 --macho-segment-name NODE_SEA`
   - For Linux: `npx postject claudeshell-{name} NODE_SEA_BLOB sea-prep.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6ac44c7517b7c16b6`
7. On macOS, codesign the binary: `codesign --sign - claudeshell-{name}`
8. Clean up temporary files (sea-config.json, sea-prep.blob).
9. Output the final binary path.

The script must be `chmod +x` and use `set -euo pipefail`.

IMPORTANT: Node.js SEA is available in Node 22+ (our minimum). Do NOT use `pkg` (deprecated
and incompatible with ESM). postject is the official Node.js tool for SEA injection.
  </action>

<task type="auto">
  <name>Task 2: Create GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/release.yml` with the following structure:

**Trigger:** `on: release: types: [published]`

**Jobs:**

1. **`test`** job:
   - runs-on: `ubuntu-latest`
   - Steps: checkout, setup Node 22, `npm ci`, `npm run build`, `npm test`
   - This gates all other jobs.

2. **`build-binaries`** job (needs: test):
   - Strategy matrix with 3 entries:
     - `{ os: macos-latest, platform: macos, arch: arm64 }` (Apple Silicon runners are arm64)
     - `{ os: macos-13, platform: macos, arch: x64 }` (macos-13 is Intel x64)
     - `{ os: ubuntu-latest, platform: linux, arch: x64 }`
   - Steps:
     a. Checkout code
     b. Setup Node.js 22
     c. `npm ci`
     d. Run `bash scripts/build-binaries.sh`
     e. Upload artifact using `actions/upload-artifact@v4` with name `claudeshell-${{ matrix.platform }}-${{ matrix.arch }}`

3. **`upload-release-assets`** job (needs: build-binaries):
   - runs-on: `ubuntu-latest`
   - Steps:
     a. Download all artifacts using `actions/download-artifact@v4` with `pattern: claudeshell-*`
     b. For each binary, upload to the GitHub release using:
        ```
        gh release upload ${{ github.event.release.tag_name }} ./claudeshell-*/claudeshell-* --clobber
        ```
     c. Use `GITHUB_TOKEN` for authentication: `env: GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}`

4. **`npm-publish`** job (needs: test):
   - runs-on: `ubuntu-latest`
   - Steps:
     a. Checkout code
     b. Setup Node.js 22 with `registry-url: 'https://registry.npmjs.org'`
     c. `npm ci`
     d. `npm run build`
     e. `npm publish` with `env: NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`

**Permissions:** `contents: write` (needed for release asset upload).

Use `concurrency: group: release-${{ github.ref }}, cancel-in-progress: true` to prevent
duplicate runs.

Do NOT use any third-party actions beyond the official `actions/*` ones. Use `gh` CLI
(pre-installed on runners) for release asset uploads.
  </action>
