<task type="auto">
  <name>Task 2: Create the p10k-style wizard in wizard.ts</name>
  <files>src/wizard.ts</files>
  <action>
Create src/wizard.ts with a sequential wizard that mirrors p10k's flow. Use readline.Interface for input, picocolors + raw ANSI for visual previews, process.stdout.write for all output (never console.log).

The wizard function signature:
```typescript
export async function executeWizard(rl: readline.Interface): Promise<WizardResult>
```

Where WizardResult extends ThemeResult with all new fields.

**Wizard Steps (each is a helper function):**

Step 1 — Font Detection (stepFontTest):
- Show "Does this look like a diamond (rotated square)?" with the Nerd Font diamond glyph \uE0B2
- Options: [y] Yes / [n] No
- If yes: set iconMode = 'nerd-font'. If no: go to Step 1b.

Step 1b — Unicode Test (stepUnicodeTest):
- Show "Does this look like a right-pointing arrow? \u25B6"
- If yes: iconMode = 'unicode'. If no: iconMode = 'ascii'.

Step 2 — Prompt Style (stepPromptStyle):
- Show numbered list of available templates with live previews using buildPromptFromTemplate
- Filter templates based on iconMode (hide requiresNerdFont templates if not nerd-font)
- User picks a number

Step 3 — Color Scheme (stepColorScheme):
- Show COLOR_SCHEMES with swatches (reuse the pattern from executeColorSelection in builtins.ts)
- User picks a number

Step 4 — Show Current Time? (stepTimeFormat):
- [1] No
- [2] 12-hour (1:23 PM)
- [3] 24-hour (13:23)

Step 5 — Separator Style (stepSeparatorStyle, only if template uses powerline separators):
- Show each separator style with a visual preview segment
- [1] Angled \uE0B0 (default)
- [2] Vertical |
- [3] Slanted \uE0BC
- [4] Round \uE0B4

Step 6 — Prompt Head/Tail Style (stepHeadStyle, only if template uses powerline):
- [1] Sharp (flat edge)
- [2] Blurred
- [3] Slanted
- [4] Round

Step 7 — Prompt Height (stepHeight):
- [1] One line
- [2] Two lines
- Show preview of each

Step 8 — Prompt Spacing (stepSpacing):
- [1] Compact (no extra newline)
- [2] Sparse (blank line before prompt)

Step 9 — Icon Density (stepIconDensity, skip if ascii):
- [1] Few icons (only essential: git branch, errors)
- [2] Many icons (all segments get icons)

Step 10 — Prompt Flow (stepFlow):
- [1] Concise (short segment labels)
- [2] Fluent (verbose labels like "on branch main")

Step 11 — Transient Prompt (stepTransient):
- [1] No — keep full prompt for past commands
- [2] Yes — show simplified prompt for history

Step 12 — Confirmation (stepConfirm):
- Show full prompt preview with ALL selected options applied
- [y] Apply / [n] Cancel
- If cancel, return empty result

**Implementation patterns:**
- Each step function takes (rl, currentConfig) and returns updated config (immutable spread)
- Use a helper `askChoice(rl, prompt, options)` that handles input validation and returns the selected option index
- Wrap everything so if user types 'q' at any step, wizard exits gracefully
- At the end, call saveConfig() with spread of loadConfig() + all new fields
- Return WizardResult with all fields set

**Visual preview helper:**
Create a `renderPreviewPrompt(config: WizardConfig)` function that builds a sample prompt line using the wizard's current config choices, used in confirmation step and for step previews.
